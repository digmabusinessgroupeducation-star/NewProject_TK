Регистр накопления "Заказы покупателей"

Назначение

Для учета заказов покупателей служит одноименный регистр (небалансовый). Смысл остатков на регистре следующий: это остатки невыполненных заказов покупателей. Отрицательные остатки на регистре не имеют смысла.

 

Измерения

  • Контрагент. Ссылается на справочники: «Контрагенты», «Подразделения компании». Контрагент, разместивший заказ.
  • Заказ. Ссылается на документ «Заказ внутренний», Ссылается на документ «Заказ покупателя». Документ оформления заказа.
  • Номенклатура. Ссылается на справочник «Номенклатура». Номенклатура заказа.
  • Характеристика номенклатуры. Ссылается на справочник «Характеристики номенклатуры». Характеристика номенклатуры заказа.
  • Склад компании. Ссылается на справочник «Склады (места хранения) компании». Склад компании, на котором был зарезервирован товар для этого заказа.

Ресурсы

  • Заказано. Тип значения: «Число». Количество номенклатуры по заказу.
  • Резерв. Тип значения: «Число». Количество номенклатуры, зарезервированноий на складе для этого заказа.

Реквизиты

  • Хозяйственная операция. Ссылается на справочник «Хоз. операции». Хозяйственная операция.
  • Сумма в валюте управленческого учаета. Тип значения: «Число». Сумма заказанной номенклатуры в валюте управленческого учета.

Дополнительная информация

Из меню "Перейти" формы документа возможен просмотр движений, сформированных этим документом.

Документы, порождающие движения: «Заказ внутренний», «Заказ покупателя», «Корректировка заказа покупателя», «Перемещение товаров», «Поступление товаров», «Реализация товаров», «Резервирование заказов покупателя», «Снятие резервов заказов покупателя».
